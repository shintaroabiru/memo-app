# UIデザイン（Pencil）

本ディレクトリは、メモアプリのUIデザイン成果物を管理する。
デザインは **Pencil** を使用して作成し、ドキュメント駆動開発の一環として `docs/` 配下に配置する。

- **デザイン方針・トークン・画面別指針**: [`design.md`](./design.md)
- **モックアップファイル**: [`design.pen`](./design.pen)
- **本ドキュメント**: ファイル配置・編集ワークフロー等の運用ルール

---

## 1. Pencil とは

[Pencil Project](https://pencil.evolus.vn/) は、オープンソースのGUIプロトタイピング・ワイヤーフレームツール。

- ワイヤーフレーム / モックアップを軽量に作成できる
- 成果物はファイルとして保存され、Gitで差分管理可能（ただしバイナリ寄りなのでマージは困難）
- 画像（PNG / SVG）へのエクスポートに対応

---

## 2. ファイル構成

```
docs/design/
├── README.md          # 本ファイル（運用ルール）
├── design.md          # デザイン方針・トークン・画面別指針
├── design.pen         # メインのモックアップ（全画面集約）
└── assets/            # エクスポートしたPNG等（必要になったら作成）
```

### 2.1 ファイル分割方針

- **初期は `design.pen` 1ファイルに全画面を集約する**
- 画面数が増えて1ファイルが扱いにくくなった場合は、画面単位で分割する
  - 例: `memo-list.pen` / `memo-edit.pen` / `tag-manage-modal.pen` / `profile.pen`
- 分割した場合は本READMEのファイル構成セクションを更新する

## 3. デザイン対象画面（フェーズ1）

`requirements.md` §5 に対応する以下4画面をモックアップする。

| 画面             | パス              | 概要                          |
|------------------|-------------------|-------------------------------|
| メモ一覧         | `/`               | 検索バー・タグフィルタ・一覧    |
| メモ新規作成     | `/memos/new`      | 新規作成フォーム                |
| メモ詳細・編集   | `/memos/[id]`     | 詳細表示と編集                  |
| プロフィール編集 | `/profile`        | プロフィール編集フォーム        |

加えて、以下のモーダル/コンポーネントもデザイン対象。

- タグ管理モーダル（タグの新規作成・編集・削除）
- メモ削除確認ダイアログ
- ピン留めトグル
- タグ選択UI（メモ作成・編集時）

## 4. 運用ルール

### 4.1 ドキュメント駆動の位置づけ

- **デザイン変更 → 実装** の順で進める（コード先行を避ける）
- 仕様（`requirements.md`）とデザイン（`design.pen`）が食い違う場合は、原則どちらかを正として揃える（先に決めたほうを正とし、もう一方を更新）

### 4.2 編集ワークフロー

1. Pencil で `docs/design/design.pen` を開く
2. 変更を加えて保存
3. 必要に応じて主要画面を PNG にエクスポートし `docs/design/assets/` に配置
4. Git にコミット
   - コミットメッセージ例: `docs: メモ一覧画面のレイアウトを更新`
   - 規約: `docs:` プレフィックス（CLAUDE.md §7.2 参照）

### 4.3 レビューのしやすさ

- `.pen` ファイルはバイナリ寄りで、PRレビュー時に差分が読めない
- **重要な変更時は、変更前後のスクリーンショット（PNG）を `assets/` にコミット**し、PR本文に貼る運用を推奨

### 4.4 命名規約

- ファイル名: kebab-case（`memo-list.pen` など）
- Pencil内のページ名: 画面名と一致させる（例: 「メモ一覧」「メモ新規作成」）

---

## 5. 実装との対応

- フロント実装時は `features/{機能名}/components/` のコンポーネントを Pencil のモックアップに合わせて作る
- Tailwind CSS のクラス設計時は、[`design.md`](./design.md) のデザイントークン（カラー・タイポグラフィ・スペーシング）を一次ソースとする
- Pencil 上の余白・配色は参考扱いとし、ピクセルパーフェクトは要求しない
- UI実装には **shadcn/ui**（`frontend/src/components/ui/` に取り込み） + **lucide-react** を採用する（詳細は [`design.md`](./design.md) §4）
