# UIデザイン方針（design.md）

本ドキュメントは、メモアプリの UI デザインの **方針・デザイントークン・画面別指針** を定義する。
Pencil で `design.pen` を作成する際の判断基準であり、Tailwind CSS 実装時のリファレンスでもある。

- 運用ルール（ファイル配置・編集ワークフロー）: [README.md](./README.md)
- 機能要件: [../requirements.md](../requirements.md)
- アーキテクチャ: [../architecture.md](../architecture.md)

> ドキュメント駆動の原則に従い、デザインの方向性に変更があれば**先に本ドキュメントを更新**してから `design.pen` / 実装に反映する。

---

## 1. デザイン原則

### 1.1 ムード

「**ミニマル & 落ち着き**」を基調とする。知的で静かな印象を参考にする。

- 余白を多めに取り、情報の塊どうしを呼吸させる
- 装飾よりもコンテンツ（メモ本文・タイトル・タグ）を主役にする
- カラーは抑制的に使い、状態の差分や重要度を**色の強度**で表現する
- 派手なアニメーションや過度な装飾は避け、トランジションは短く（150〜200ms）

### 1.2 守るべき体験

1. **書くことに集中できる**: 入力欄は広く、周辺ノイズを最小化する
2. **一覧で目的のメモにすぐ辿り着ける**: 検索・タグ・ピン留めで素早く絞り込める
3. **操作の見通しがよい**: ボタンの役割が一目で分かり、破壊的操作は確認を挟む

## 2. デザイントークン

### 2.1 カラー

#### 2.1.1 ニュートラル（ベース）

Tailwind の **Slate**（青みがかったグレー）を採用する。クールで明瞭な雰囲気を狙う。

| トークン         | 役割                           | Tailwind 参照 |
| ---------------- | ------------------------------ | ------------- |
| `bg-base`        | アプリ全体の背景               | `slate-50`    |
| `bg-surface`     | カード・モーダル・ヘッダーの面 | `white`       |
| `bg-subtle`      | ハイライト・ホバー時の背景     | `slate-100`   |
| `border-default` | カード・入力欄の枠線           | `slate-200`   |
| `border-strong`  | フォーカス時・選択中の枠線     | `slate-300`   |
| `text-primary`   | 本文・タイトルの主テキスト     | `slate-900`   |
| `text-secondary` | キャプション・補助テキスト     | `slate-600`   |
| `text-muted`     | 無効状態・プレースホルダ       | `slate-400`   |

> ダークモードはフェーズ1ではスコープ外。実装時も `dark:` クラスは付与しない。将来導入する場合は本セクションを拡張する。

#### 2.1.2 アクセント（プライマリ）

Tailwind の **Teal**（エメラルドグリーン寄り）を採用する。落ち着きと「証」「保存される」イメージを両立する色味。

| トークン             | 役割                                     | Tailwind 参照 |
| -------------------- | ---------------------------------------- | ------------- |
| `accent-default`     | プライマリボタン背景・リンク・チェック等 | `teal-600`    |
| `accent-hover`       | プライマリボタンのホバー                 | `teal-700`    |
| `accent-active`      | プライマリボタンの押下                   | `teal-800`    |
| `accent-soft-bg`     | 選択中のタグチップ・トースト背景の薄塗り | `teal-50`     |
| `accent-soft-border` | 選択中のタグチップ枠線                   | `teal-200`    |
| `accent-soft-text`   | 薄塗り背景上のテキスト                   | `teal-700`    |
| `focus-ring`         | フォーカスリング                         | `teal-500`    |

#### 2.1.3 セマンティック

| トークン  | 用途                                       | Tailwind 参照 |
| --------- | ------------------------------------------ | ------------- |
| `success` | 保存完了トースト、成功表示                 | `emerald-600` |
| `warning` | 注意喚起・上限近接など                     | `amber-500`   |
| `danger`  | 削除ボタン背景・エラーメッセージのアイコン | `rose-600`    |
| `info`    | 補助情報・チップ                           | `sky-600`     |

#### 2.1.4 ピン留めアイコン

ピン留めは「お気に入り」のメタファとして **amber-500（黄味のあるアンバー）** を使用する。アクセントカラー（teal）と衝突せず、リストの中で素早く識別できる。

### 2.2 タイポグラフィ

#### 2.2.1 フォントスタック

```
font-sans: 'Inter', 'Noto Sans JP', system-ui, -apple-system, 'Hiragino Sans',
           'Yu Gothic', sans-serif
```

- 英数字・記号は **Inter**、日本語は **Noto Sans JP** を Web フォントとして読み込む
- フォールバックとして OS のシステムフォントを並べる
- 等幅は当面想定しない（メモはプレーンテキスト前提。コードブロック表示はフェーズ1スコープ外）
- Web フォントの読み込み方法（next/font 推奨）は実装フェーズで確定する

#### 2.2.2 サイズスケール

| トークン    | サイズ / 行高 | 用途                         |
| ----------- | ------------- | ---------------------------- |
| `text-xs`   | 12px / 16px   | キャプション・タイムスタンプ |
| `text-sm`   | 14px / 20px   | 補助テキスト・タグチップ     |
| `text-base` | 16px / 24px   | 本文・入力欄                 |
| `text-lg`   | 18px / 28px   | カードのメモタイトル         |
| `text-xl`   | 20px / 28px   | セクション見出し             |
| `text-2xl`  | 24px / 32px   | ページタイトル               |

- 行高は読みやすさ優先でやや広め
- 文字間隔（letter-spacing）はデフォルトのまま

#### 2.2.3 ウェイト

- `font-normal` (400): 本文
- `font-medium` (500): ボタンラベル・タグチップ・補助強調
- `font-semibold` (600): 見出し・カードタイトル
- `font-bold` (700) は基本使わない（ミニマル基調のため）

### 2.3 スペーシング

Tailwind のデフォルトスケール（`4px` 刻み）を使う。デザイン上の基準は以下。

| 用途                         | 基準値   | Tailwind  |
| ---------------------------- | -------- | --------- |
| アイコンとテキストの間       | 8px      | `gap-2`   |
| フォーム要素どうしの縦間隔   | 16px     | `gap-4`   |
| セクション間の縦間隔         | 24〜32px | `gap-6/8` |
| ページの内側余白（PC）       | 32px     | `p-8`     |
| ページの内側余白（モバイル） | 16px     | `p-4`     |
| カード内側の余白             | 16px     | `p-4`     |
| モーダル内側の余白           | 24px     | `p-6`     |

### 2.4 角丸

落ち着いた印象のため、**やや控えめ**の角丸を使う。

| トークン       | 値     | 用途                         |
| -------------- | ------ | ---------------------------- |
| `rounded`      | 4px    | タグチップ・小さなバッジ     |
| `rounded-md`   | 6px    | ボタン・入力欄               |
| `rounded-lg`   | 8px    | カード・モーダル             |
| `rounded-full` | 9999px | アバター・丸型アイコンボタン |

### 2.5 シャドウ

ミニマル基調のため、シャドウはほぼ使わず**境界線で要素を区切る**方針。例外として、以下のみシャドウを許容する。

| 用途           | Tailwind    |
| -------------- | ----------- |
| モーダル       | `shadow-xl` |
| ドロップダウン | `shadow-lg` |
| トースト       | `shadow-md` |

通常のカードは `border border-slate-200` のみで区切る。

### 2.6 トランジション

- 既定: `transition duration-150 ease-out`
- ボタン・ホバー時の背景色変化: 150ms
- モーダルのフェード/スライド: 200ms
- 派手なアニメーションは避ける

## 3. レイアウト方針

### 3.1 ブレークポイント

Tailwind のデフォルトに従い、**レスポンシブ重視**で3層を丁寧に設計する。

| 名前     | 幅 (min) | 想定デバイス   |
| -------- | -------- | -------------- |
| （base） | 0        | モバイル       |
| `sm`     | 640px    | 大きめモバイル |
| `md`     | 768px    | タブレット     |
| `lg`     | 1024px   | ノートPC       |
| `xl`     | 1280px   | デスクトップ   |

Pencil 上では **モバイル（375px）・タブレット（768px）・PC（1280px）** の3サイズで主要画面のワイヤーを作成する。

### 3.2 ナビゲーション構造

フェーズ1は画面数が少ないため、ナビゲーションは軽量に保つ。

- **PC（`lg` 以上）**:
  - 左サイドバーにグローバルナビ（メモ一覧・タグ管理ボタン・プロフィール）
  - 検索バーはサイドバー上部 or メインエリア上部に固定
- **タブレット・モバイル（`lg` 未満）**:
  - 上部ヘッダーにロゴと検索アイコン
  - 画面下部にボトムバー（メモ一覧 / 新規作成 / プロフィール の3タブ）
  - タグ管理はメモ一覧画面内のボタンから到達

> ナビ構造は実装段階で再検証する。サイドバー vs ボトムバーの最終決定は Pencil でワイヤーを比較してから行う。

### 3.3 コンテンツ最大幅

- メモ詳細・編集画面の本文エリア: 最大 720px（読みやすさ優先）
- メモ一覧画面: コンテナ最大 1200px

## 4. コンポーネント方針

### 4.1 ライブラリ採用

#### shadcn/ui を採用する（2026-05-23 確定）

- Radix UI ベースのアクセシビリティ済みコンポーネント群を、ソースコードとして直接プロジェクト内に取り込む方式
- モーダル / ドロップダウン / トースト / ポップオーバー / コンボボックスといった「自作するとコストの高い部品」の実装負荷を下げる
- スタイルは Tailwind で完全に上書きできるため、本ドキュメントのデザイントークンに揃えやすい
- CLAUDE.md §3「上記以外のライブラリ・フレームワークを導入する場合は、必ず事前に確認を取ること」に該当する追加導入として、**2026-05-23 にユーザー承認済み**
- 配置先: `frontend/src/components/ui/`（shadcn CLI の出力先と一致）。ファイルはプロジェクトに直接コピーされるため、必要に応じて Tailwind クラスでデザイントークンに合わせて上書きする

### 4.2 アイコン

- **lucide-react** を採用する（shadcn/ui の既定アイコンライブラリ）。shadcn/ui 採用承認に内包される形で導入する
- ピン留め・タグ・検索・削除・編集・チェック・閉じる、など最小限のアイコンに留める
- サイズは原則 `16px` / `20px` / `24px` の3段階

### 4.3 主要コンポーネントの方針

| コンポーネント      | 方針                                                                                |
| ------------------- | ----------------------------------------------------------------------------------- |
| ボタン（Primary）   | 背景 `accent-default` / テキスト白 / `rounded-md` / `px-4 py-2` / `font-medium`     |
| ボタン（Secondary） | 背景 `white` / 枠線 `border-default` / テキスト `text-primary`                      |
| ボタン（Danger）    | 背景 `danger` / テキスト白。削除確認ダイアログ内の確定ボタン専用                    |
| ボタン（Ghost）     | 背景なし / ホバーで `bg-subtle`。アイコンボタンやサブ操作用                         |
| 入力欄（Input）     | `bg-surface` / `border-default` / フォーカスで `focus-ring` を太め (2px) で表示     |
| テキストエリア      | 入力欄と同様。最小高さ 200px、自動高さ拡張は検討                                    |
| タグチップ          | 角丸 `rounded` / `text-sm` / 枠線 `border-default`。選択中は `accent-soft-*` を適用 |
| カード              | `bg-surface` / `border-default` / `rounded-lg` / `p-4`                              |
| モーダル            | `shadow-xl` / `rounded-lg` / 中央配置 / 背景は `bg-slate-900/40` のオーバーレイ     |
| トースト            | 右下 (PC) / 上部 (モバイル) / `shadow-md` / 4秒で自動クローズ                       |
| 確認ダイアログ      | モーダルの一種。タイトル + 説明文 + キャンセル/確定ボタン                           |
| ピン留めトグル      | アイコンボタン。ピン留め時は `amber-500`、未ピン時は `text-muted`                   |

### 4.4 状態表現

| 状態         | 表現                                                |
| ------------ | --------------------------------------------------- |
| 通常         | デザイントークン通り                                |
| ホバー       | 背景を1段濃く（例: `bg-surface` → `bg-subtle`）     |
| フォーカス   | `focus-ring` で 2px のリングを表示                  |
| アクティブ   | プライマリボタンは `accent-active` で1段濃く        |
| 無効         | `text-muted` / `bg-subtle` / `cursor-not-allowed`   |
| ローディング | スピナー or スケルトン。ボタンは "保存中..." 表記   |
| エラー       | 入力欄下に `text-sm text-rose-600` でメッセージ表示 |

## 5. 画面別デザイン指針

`requirements.md` §5 で定義された画面およびモーダルについて、Pencil 上のワイヤーフレーム作成時の指針を示す。

### 5.1 メモ一覧（`/`）

#### 共通要素

- 上部に検索バー（プレースホルダ: 「メモを検索...」）
- 検索バー右隣に「タグフィルタ」ドロップダウン（複数選択、AND条件）
- 「ピン留めのみ表示」のトグル
- 「タグ管理」ボタン（モーダルを開く）
- 「新規作成」ボタン（PCでは右上に固定、モバイルではボトムバーの中央 + のアイコン）
- 空状態（メモ0件）には説明テキストと「最初のメモを作成」CTA を表示
- 検索結果0件の状態は空状態と区別して「該当するメモがありません」表示

#### レイアウト案：両案を Pencil 上で作成して比較する

**案A: カードグリッド（Google Keep 風）**

- カード内: タイトル（`text-lg font-semibold` / 1行truncate） / 本文プレビュー（`text-sm text-secondary` / 4行 line-clamp） / タグチップ（最大3個 + `+N`） / 更新日時 + ピン留めアイコン
- レスポンシブ:
  - モバイル: 1列
  - タブレット (`md`): 2列
  - PC (`lg`): 3列
- メリット: 視覚的に把握しやすい / 本文プレビューが多く取れる
- デメリット: 1画面あたりの件数が少なくなる

**案B: 縦長リスト（Apple Notes 風）**

- 1行 = 1メモ。タイトル（左） / 本文プレビュー（1行 truncate） / タグ（1〜2個） / 更新日時（右） / ピン留めアイコン（右端）
- レスポンシブ:
  - モバイル: タイトル + 本文プレビュー + ピン留めアイコン（タグと日時は省略 or 縮小）
  - PC: 全要素を1行に並べる
- メリット: 件数が多くても一覧性が高い / 検索結果との相性が良い
- デメリット: 本文プレビューが短くなる

> 両案を `design.pen` 上に並べ、Pencil 上で見比べてから採用案を決める。決定後は不採用案を削除して本ドキュメントを更新する。

#### ソート順

- デフォルト: `is_pinned` 降順 → `updated_at` 降順（`requirements.md` §3.4 準拠）
- ピン留めメモはセクション見出し「ピン留め」とともに上部に固める案も検討する

### 5.2 メモ新規作成（`/memos/new`）

- タイトル入力欄（1行 / `text-xl` / プレースホルダ: 「タイトル」）
- 本文入力欄（textarea / `text-base` / プレースホルダ: 「本文を入力...」 / 最小高さ 320px）
- タグ選択UI（既存タグから選択）+ 「タグを新規作成」ボタン（タグ管理モーダルを開く）
- ピン留めトグル（ヘッダー右側にアイコンで配置）
- 保存ボタン（プライマリ） / キャンセルボタン（Ghost）
- バリデーションエラーは各入力欄の直下に表示
- 文字数カウンター: タイトル（残り文字数 / 100）、本文（残り文字数 / 10,000）

### 5.3 メモ詳細・編集（`/memos/[id]`）

- 表示と編集は **同一画面・編集モード切り替え** で実装する
  - 初期表示は閲覧モード
  - タイトル or 本文をクリック / 編集ボタン押下で編集モードに入る
- 編集モードのレイアウトは 5.2 と統一する
- 削除ボタンは画面右上の「…」メニュー内に配置（押下で削除確認ダイアログ）
- ピン留めトグルはヘッダー右側のアイコン
- 編集中の未保存変更がある状態でナビゲーションを試みた場合は確認ダイアログを表示

### 5.4 プロフィール編集（`/profile`）

- アバター画像プレビュー（丸型 / 96px） + アバターURL入力欄
- 表示名入力欄（`min_length=1, max_length=50`）
- 自己紹介入力欄（textarea / `max_length=200` / 任意）
- 保存ボタン / キャンセルボタン
- 保存成功時はトーストで通知
- 空文字列の bio / avatar_url は「未設定」と表示する

### 5.5 タグ管理モーダル

- モーダルタイトル: 「タグの管理」
- タグ一覧（リスト表示）。各行に「リネーム」「削除」アクション
- 上部に「新規タグを追加」入力欄 + 追加ボタン
- リネームはインライン編集（クリックで `<input>` に切り替え）
- 削除確認は inline で「本当に削除しますか？」を表示（このタグが N 件のメモに紐付いていることを警告）
- 1〜20文字、ユーザー内ユニーク（`requirements.md` §3.7 準拠）
- バリデーションエラーは入力欄の直下に表示

### 5.6 削除確認ダイアログ

- タイトル: 「メモを削除しますか？」
- 説明: 「この操作は取り消せません」
- ボタン: 「キャンセル」（Secondary） / 「削除する」（Danger）
- フォーカスは初期状態でキャンセルに当てる（誤操作防止）
- Esc キーでキャンセル

### 5.7 タグ選択UI（メモ作成・編集時）

- コンボボックス形式（shadcn/ui の Command コンポーネント想定）
- 入力でタグを絞り込み、クリックで追加
- 選択済みタグはチップとして表示し、× で削除
- 「タグを新規作成」ボタンを末尾に配置（タグ管理モーダルを開く）
- 1メモあたり最大10個（超過時は追加ボタンを無効化）

## 6. アクセシビリティ

`requirements.md` §4 で「最低限のキーボード操作・コントラスト確保」と定義されているため、以下を最低ラインとする。

- すべてのインタラクティブ要素はキーボード操作可能（Tab / Shift+Tab / Enter / Space / Esc）
- フォーカスインジケータは省略しない（`focus-ring` を必ず可視化）
- フォーム要素には `<label>` を必ず紐付ける
- アイコンのみのボタンには `aria-label` を付ける
- コントラスト比は WCAG AA 相当（本文 4.5:1 以上）を満たす
- モーダル表示中はフォーカスをトラップし、Esc で閉じる
- 削除等の破壊的操作は色だけでなくテキストでも明示する
