# メモアプリ開発プロジェクト

## 📌 プロジェクト概要

プログラミングスキル向上を目的とした**メモアプリ**を開発する。
本ドキュメントは、AI駆動開発およびドキュメント駆動開発の起点として活用する。

---

## 🛠 技術スタック

### フロントエンド

- **Next.js**（App Router）
- **TypeScript**（strict mode）
- **React**
- **Zod**（バリデーション）
- **Zustand**（状態管理）
- **Tailwind CSS**
- **Vitest**（テストランナー）
- **ESLint + Prettier**（Lint / Formatter）

### バックエンド

- **FastAPI**
- **Python**（3.11以上）
- **PostgreSQL**
- **SQLAlchemy** / **Alembic** / **Pydantic**
- **pytest**（テスト）
- **Ruff**（Lint / Formatter）
- **Docker**（環境構築に使用）

### デザイン

- **Pencil**（UIモックアップ・ワイヤーフレーム）
  - 成果物: `docs/design/design.pen`

### 開発環境

- **エディタ**: Cursor
- **AIアシスタント**: Claude Code

---

## 🚀 開発フェーズ

開発は2つのフェーズに分けて段階的に進める。

### フェーズ1：コア機能の実装

メモアプリとしての基本機能を最優先で構築する。

- メモの**新規作成**
- メモの**編集**
- メモの**削除**
- メモの**一覧表示**
- メモの**検索**
- ユーザープロフィールの**編集**

### フェーズ2：認証機能の実装

認証周りは複雑なため、フェーズ1の基盤が固まった後に着手する。

- ユーザーの**新規作成**
- **認証機能**（ログイン等）

---

## 📝 補足

- フェーズ1ではまず認証なしでアプリを動作させ、メモCRUDと検索を中心にコア体験を完成させる方針。
- フェーズ2で認証を後付けする設計を見据えて、フェーズ1から「将来的にユーザー単位でデータを扱う」前提でDBスキーマを設計しておくとスムーズに移行できる。
